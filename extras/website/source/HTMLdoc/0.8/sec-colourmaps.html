1.12 Colour Maps

Colour maps provide a graphical means of producing two-dimensional representations of surfaces, or equivalently of producing maps of the values of functions of two variables. Each point in the plane is assigned a colour which indicates the value associated with that point. In the following Section, we refer to the third coordinate as rather than , to distinguish it from the third axes of three-dimensional plots1.

The default mapping used between values of and colour is a greyscale mapping, such that the smallest value of in any given colour map corresponds to black, and largest value corresponds to white. More generally, however, the user can supply any algebraic expressions for the RGB, HSB or CMYK components of the colours to be used, and even specify a mask such that some parts of the colour map appear transparent. Moreover, these custom colour mappings need not be one-parameter mappings depending only upon a single variable , but can depend on up to four quantities , , and .

If data is read from a datafile, then the first two columns of output data are assumed to contain the respective positions of each datapoint along the -axis and the -axis. The next column contains the value , and may be followed by up to three further optional values , and . In the case where one or more function(s) are supplied, they are assumed to be functions of both and , and are sampled at a grid of points in the plane; the first supplied function returns the value , and may be followed by up to three further optional functions.

In the following simple example, a colour map of the complex argument of the Riemann zeta function is produced, taking the plane to be an Argand plane in which each point corresponds to a value of , with being the real axis, and being the imaginary axis:

set nokey
set size 8 square
set samples grid 400x400
set c1range[-pi:pi]
